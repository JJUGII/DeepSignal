# DeepSignal 공격형 엔진 개발 기획서 — 숏·레버리지·고속 회전

> 목표: "리스크를 키운 만큼 수익 폭도 커지는" 비대칭 해소.
> 핵심 원칙: **엣지 먼저, 공격성은 그다음.** 검증 안 된 전략은 빈도·레버리지를 올려도 손실만 빨라진다.
> 작성 기준: 2026-06 현재 코드.

---

## 0. 대전제 (이걸 어기면 다 무의미)

1. **모든 신규 전략은 EDGE_GATE에 등록 → 백테스트 검증 통과(연속 검증)해야 라이브.**
   레버리지·고속은 "이기는 전략"을 **증폭**하는 도구일 뿐, 없는 엣지를 만들지 못한다.
2. **레버리지는 손실도 2~3배.** 안전장치(손실한도·포지션캡·트레일링스톱)를 먼저 강화하고 기능을 켠다.
3. 한국 개인은 개별주식 공매도가 어렵다 → **"숏 = 인버스 ETF 매수", "레버리지 = 레버리지 ETF 매수"**로 현물 경로에서 합법·간단하게 구현. (선물·해외거래소는 후순위 옵션)

---

## 1. 숏 (하락장 수익) — 인버스 ETF

### 작동 방식
- 하락 신호 감지 → **인버스 ETF를 매수**(= 사실상 숏). 지수가 내리면 ETF가 오른다.
- 종목(현물·KIS 매수만으로 구현):
  - 국내: `114800 KODEX 인버스`(-1x), `252670 KODEX 200선물인버스2X`(-2x)
  - 해외: `SQQQ`(나스닥 -3x), `SH/SDS`(S&P -1x/-2x)
- **추세추종 확장**: 현재 S&P500 < 200일선이면 "현금"인데 → 옵션으로 **인버스 보유**까지 허용(하락장에서도 수익 시도).

### 개발
- `deepsignal/live_trading/regime_trend.py` 확장: `decide_regime_trend`에 `out_market_action = cash | inverse` 옵션 추가. out이면 인버스 ETF ENTER.
- 신규 `deepsignal/strategy/inverse_signal.py`: 하락 추세·과열 반전 신호로 인버스 진입/청산 판단.
- EDGE_GATE 전략 등록: `inverse_kr`, `inverse_us`.

---

## 2. 레버리지 (상승장 증폭) — 레버리지 ETF

### 작동 방식
- 강한 상승·고확신 신호 → 일반 ETF 대신 **레버리지 ETF 매수**로 같은 자본에 2~3배 노출.
  - 국내: `122630 KODEX 레버리지`(+2x)
  - 해외: `TQQQ`(나스닥 +3x), `SOXL`(반도체 +3x), `SPXL`(S&P +3x)
- 신호 강도(GSQS·P(win))에 따라 **일반 → 2x → 3x 단계적 노출**.

### 개발
- `deepsignal/risk/leverage_policy.py` 신규: 신호강도 → 레버리지 배수·종목 매핑, 변동성 기반 포지션 축소.
- 주문 경로는 기존 KIS ETF 매수 그대로 재사용(현물이라 추가 API 불필요).
- ⚠️ **추적오차·복리감쇠(volatility decay)** 주의: 레버리지 ETF는 횡보장에서 갉아먹힘 → "단기·추세 확인 구간"에서만 사용하도록 보유기간 제한.

---

## 3. 고속 회전 (장중 연속 매매)

### 현재 한계
- 코인: 러너 **1분 틱** (`--interval-minutes 1.0`)
- 국내주식: **하루 1회(09:05) 플랜** → 장중 회전 사실상 없음

### 작동 방식 (목표)
- **코인**: 틱 주기 단축(예 15~30초) + **분봉(이미 차트용으로 추가됨) 기반** 단기 신호. 종목당 쿨다운으로 과매매 방지.
- **국내·해외주식**: "하루 1회 플랜" → **장중 N분 간격 연속 스캔·매매 루프**로 전환. 보유 종목은 매 틱 TP/SL·트레일링스톱 점검.

### 개발
- `deepsignal/live_trading/runner/intraday_runner.py` 신규: 장중 루프(예 1~3분 간격) — 스캔 → 신호 → 무승인 체결 → 보유 점검.
- 코인 러너 주기 파라미터화·하향, 신호 쿨다운·시간당 한도로 과매매 차단.
- **KIS 초당 요청 한도(EGW00201) 대응**: 요청 큐·레이트리미터(초당 N건 제한) 도입 — 고속화의 필수 전제.

## 4. 리스크 관리 강화 (기능보다 먼저)

| 항목 | 강화 내용 |
|---|---|
| 트레일링 스톱 | 고점 대비 X% 하락 시 자동 청산 (레버리지·고속 필수) |
| 일일 손실 한도 | 레버리지 사용 시 한도 자동 축소(변동성 비례) |
| 포지션 캡 | 레버리지 ETF는 1회·총액 캡을 일반의 절반로 |
| 보유기간 제한 | 레버리지/인버스 ETF는 추세 유효 구간만, 횡보 N일이면 강제 정리 |
| 서킷브레이커 | 단시간 급락 감지 시 신규 진입 전면 중단(기존 킬스위치 확장) |

파일: `deepsignal/risk/trading_halt.py`(확장), `deepsignal/risk/dynamic_tpsl.py`(트레일링), 신규 `deepsignal/risk/rate_limiter.py`.

## 5. 개발 로드맵 (단계별 — 검증 게이트 통과해야 다음으로)

| Phase | 내용 | 산출물 |
|---|---|---|
| **P0. 백테스트 인프라** | 인버스·레버리지·고속 전략을 과거 데이터로 검증할 틀 (생존편향·비용·추적오차 반영) | edge_monitor에 신규 전략 등록 |
| **P1. 리스크 먼저** | 트레일링스톱·레이트리미터·레버리지용 한도 축소 | risk 모듈 |
| **P2. 인버스(숏)** | 추세추종 out→인버스 + 인버스 신호, **백테스트 검증** | regime_trend 확장, inverse_signal |
| **P3. 레버리지** | 신호강도별 레버리지 ETF, 보유기간 제한, **검증** | leverage_policy |
| **P4. 고속 회전** | 장중 인트라데이 루프(주식) + 코인 주기 단축, 레이트리미터 | intraday_runner |
| **P5. (선택) 선물** | Binance 선물(코인 진짜 숏·고배율) — 대규모 추가·고위험, 별도 검토 | 신규 브로커 |

각 Phase는 **EDGE_GATE 검증 통과 시에만** 라이브 전환. 미검증이면 dry-run/페이퍼로만.

---

## 6. 공격성 단계와의 연결
- P2 완료 → 하락장 수익 가능(공격성 6~7)
- P3 완료 → 레버리지 노출(공격성 8)
- P4 완료 → 고속 회전·고빈도(공격성 8~9)
- P5(선물) → 공격성 9~10 (청산 위험 포함)

---

## 7. 솔직한 경고
- **레버리지 ETF는 장기 보유 시 원금이 녹습니다**(복리감쇠). 반드시 단기·추세구간 한정.
- **고빈도 = 비용 폭증**: 엣지 없이는 수수료·슬리피지로 손실 가속. P0 백테스트에서 비용 반영 후 양(+)이어야만 진행.
- **선물(P5)**은 청산(원금 전액 손실) 위험 + 거래소 가입·세금·규제 이슈 → 가장 마지막, 가장 신중히.
- 타인에게 제공·구독화하면 자본시장법(투자일임) 이슈는 별개로 여전히 적용.
