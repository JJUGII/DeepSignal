# DeepSignal — 개발 규칙

Cursor 및 인간 개발자 모두에게 동일하게 적용한다.

## 안전·보안

1. **실전 자동매매** 기능은 **검증 전에 구현하지 않는다.**  
   - 검증: 최소한 백테스트 또는 문서화된 재현 절차 + 모의투자 경로가 갖춰진 상태를 말한다.  
2. **API 키·비밀번호·토큰을 코드에 하드코딩하지 않는다.**  
3. **모든 외부 API 키는 `.env`로 관리**한다.  
   - `.env`는 버전 관리에 포함하지 않는다.  
   - `.env.example`에는 키 이름만 적고 값은 비운다.  

## 설계·실행 환경

4. **처음에는 외부 API 없이도 로컬에서 실행 가능한 구조**를 우선한다 (샘플 데이터, SQLite, 오프라인 플래그 등).  
5. **Windows 환경에서 실행 가능**해야 한다 (경로 구분자, 인코딩, 콘솔 인자 등 고려).  

## 품질·검증

6. **기능을 추가할 때 테스트 파일도 함께 추가**한다 (단위 테스트 또는 스모크 테스트).  
7. **수익률보다 리스크 관리와 재현성**을 우선한다 (로그, 시드, 버전 고정 가능성).  
8. **백테스트 없는 전략은 실전 주문으로 연결하지 않는다.**  
9. **브로커 연동 전에 `paper_trading`과 동일한 주문·포지션 인터페이스**를 먼저 완성한다 (실전은 구현체만 교체 가능하게).
